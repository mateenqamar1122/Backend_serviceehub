📊 TABLE RELATIONSHIPS

Supabase auth.users
    ↓ (REFERENCES)
public.users (Primary app users table)
    ├─ ONE-TO-ONE → provider_profiles
    ├─ ONE-TO-MANY → services (as provider)
    ├─ ONE-TO-MANY → bookings (as provider or customer)
    ├─ ONE-TO-MANY → messages (as sender/recipient)
    ├─ ONE-TO-MANY → reviews (as reviewer/reviewee)
    └─ ONE-TO-ONE → subscriptions (for providers)

services (Service listings)
    ├─ MANY-TO-ONE → users (provider_id)
    ├─ ONE-TO-MANY → bookings
    ├─ ONE-TO-MANY → reviews
    └─ MANY-TO-MANY → users (via provider_services)

bookings (Service orders)
    ├─ MANY-TO-ONE → services
    ├─ MANY-TO-ONE → users (provider_id)
    ├─ MANY-TO-ONE → users (customer_id)
    ├─ MANY-TO-ONE → users (cancelled_by)
    └─ ONE-TO-ONE → reviews

📊 COMMON QUERIES

Find Available Services:
──────────────────────
SELECT s.* FROM public.services s
WHERE s.status = 'active'
  AND s.is_available = true
ORDER BY s.average_rating DESC
LIMIT 20;

Get Provider Statistics:
────────────────────────
SELECT u.id, u.username, COUNT(b.id) as bookings,
       u.average_rating
FROM public.users u
LEFT JOIN public.bookings b ON u.id = b.provider_id
WHERE u.role = 'provider'
GROUP BY u.id
ORDER BY u.average_rating DESC;

Get Booking Details:
─────────────────────
SELECT b.*, s.title, u_cust.username, u_prov.username
FROM public.bookings b
JOIN public.services s ON b.service_id = s.id
JOIN public.users u_cust ON b.customer_id = u_cust.id
JOIN public.users u_prov ON b.provider_id = u_prov.id
WHERE b.id = ?;

Search Services by Keyword:
───────────────────────────
SELECT * FROM public.services
WHERE to_tsvector('english', title || ' ' || description)
      @@ plainto_tsquery('english', ?)
  AND status = 'active'
ORDER BY average_rating DESC;
